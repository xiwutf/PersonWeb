# 向量存储空间说明

## 📦 存储位置

向量数据存储在**本地文件系统**中，默认路径为：
```
./data/chroma
```

可以通过环境变量 `CHROMA_DB_PATH` 修改存储路径。

## 💾 存储内容

每个文档的向量数据包括：
1. **向量数据**：每个文档片段（chunk）的向量表示（通常是 1536 维，每个维度 4 字节）
2. **原始文本**：文档片段的完整文本内容
3. **元数据**：文档 ID、片段索引、摘要等信息

## 📊 存储空间估算

### 单个文档的存储空间

假设一个文档被分成 100 个片段（chunk）：

- **向量数据**：100 chunks × 1536 维 × 4 字节 ≈ **600 KB**
- **文本数据**：取决于文档大小，假设每个 chunk 平均 1000 字符 ≈ **100 KB**
- **元数据**：相对较小，约 **10 KB**
- **Chroma 索引开销**：约 **100 KB**

**总计**：单个文档约占用 **800 KB - 1 MB**（取决于文档大小）

### 大量文档的存储空间

- **100 个文档**：约 **80-100 MB**
- **1000 个文档**：约 **800 MB - 1 GB**
- **10000 个文档**：约 **8-10 GB**

## ⚠️ 注意事项

1. **文件越多，存储空间越大**：这是正常的，每个文档都需要存储向量和文本
2. **存储是持久的**：即使删除原始文档文件，向量数据仍然保留在数据库中
3. **需要定期清理**：删除不再需要的文档时，应该同时删除对应的向量数据

## 🧹 清理机制

### 方式 1：通过 API 删除（推荐）

调用删除接口：
```http
DELETE /api/ai/document/{document_id}/vectors
```

### 方式 2：手动删除数据库文件

如果需要完全清理，可以删除整个数据库目录：
```bash
rm -rf ./data/chroma
```

⚠️ **注意**：这会删除所有文档的向量数据！

### 方式 3：删除特定文档的集合

Chroma 为每个文档创建独立的集合，集合名称为：`document_chunks_{document_id}`

可以通过 Chroma 客户端删除特定集合。

## 🔧 优化建议

1. **定期清理**：删除不再使用的文档向量
2. **监控存储空间**：定期检查 `./data/chroma` 目录的大小
3. **使用外部存储**：如果需要，可以将 `CHROMA_DB_PATH` 指向外部存储（如网络磁盘）
4. **压缩存储**：Chroma 内部已经对数据进行了一定程度的压缩

## 📈 存储空间监控

可以通过以下命令查看存储空间使用情况：

```bash
# Windows
dir /s ./data/chroma

# Linux/Mac
du -sh ./data/chroma
```

## 🔄 迁移到云存储（可选）

如果需要将向量数据存储到云端，可以考虑：

1. **使用 Chroma 的远程客户端**：连接到远程 Chroma 服务器
2. **使用其他向量数据库**：如 Pinecone、Weaviate、Qdrant 等云服务
3. **定期备份**：将本地数据备份到云存储

## 📝 相关配置

在 `.env` 文件中可以配置：

```env
# 向量数据库存储路径
CHROMA_DB_PATH=./data/chroma

# 集合名称
CHROMA_COLLECTION_NAME=document_chunks
```
