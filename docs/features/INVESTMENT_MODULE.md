# 投资管理模块功能说明

## 📋 模块概述

投资管理模块是一个完整的个人投资组合管理系统，支持股票和基金的投资记录、价格跟踪、盈亏计算和可视化分析。

## 🎯 核心功能

### 1. 投资记录管理
- **添加投资**：支持添加股票和基金投资记录
- **编辑投资**：可以修改投资记录的代码、名称、持仓数量、成本价等信息
- **删除投资**：支持删除不需要的投资记录
- **查看详情**：查看每笔投资的详细信息

### 2. 自动数据获取
- **自动获取名称和价格**：根据代码自动从东方财富 API 获取基金/股票名称和当前价格
- **智能代码识别**：根据代码自动识别是股票还是基金
- **价格刷新**：批量刷新所有投资记录的当前价格

### 3. 数据计算
- **自动计算总成本**：总成本 = 持仓数量 × 成本价
- **自动计算市值**：市值 = 持仓数量 × 当前价格
- **自动计算盈亏**：盈亏 = 市值 - 总成本
- **自动计算收益率**：收益率 = (盈亏 / 总成本) × 100%

### 4. 数据可视化
- **资产分布图表**：显示 Top 10 持仓的资产分布（饼图）
- **收益排行图表**：显示 Top 5 收益最高的投资（柱状图）
- **按类型统计**：统计股票和基金的数量、总成本、总市值、盈亏等
- **盈利状态统计**：统计盈利和亏损的投资数量及金额

### 5. 统计面板
- **总资产统计**：显示总成本、总市值、总盈亏、总收益率
- **持仓数量统计**：显示总持仓数量
- **Top 5 持仓**：按市值排序显示前 5 名持仓
- **收益排行**：按盈亏排序显示收益最高的投资

## 🏗️ 技术架构

### 前端技术栈
- **框架**：Vue 3 + Nuxt 3
- **图表库**：ECharts
- **UI 组件**：自定义组件 + Tailwind CSS
- **状态管理**：Composition API (ref, computed, watch)

### 后端技术栈
- **框架**：ASP.NET Core Web API
- **数据库**：Entity Framework Core
- **外部 API**：东方财富 API（获取实时价格）

### 数据模型

#### Investment（投资记录）
```csharp
public class Investment
{
    public long Id { get; set; }
    public string Code { get; set; }        // 代码（如：005918）
    public string Name { get; set; }        // 名称（如：天弘沪深300ETF联接C）
    public string Type { get; set; }        // 类型（stock/fund）
    public decimal Quantity { get; set; }   // 持仓数量
    public decimal CostPrice { get; set; }  // 成本价
    public decimal CurrentPrice { get; set; } // 当前价格
    public decimal TotalCost { get; set; }  // 总成本
    public decimal MarketValue { get; set; } // 市值
    public decimal ProfitLoss { get; set; }  // 盈亏
    public decimal ProfitRate { get; set; }  // 收益率
    public string? Notes { get; set; }       // 备注
    public DateTime CreatedAt { get; set; }
    public DateTime UpdatedAt { get; set; }
}
```

## 🔌 API 接口

### 1. 获取投资列表
```
GET /api/Investment
```
**响应示例**：
```json

  "message": "success",

    {
      "id": 1,

      "name": "天弘沪深300ETF联接C",
      "type": "fund",
      "quantity": 1000,

      "totalCost": 1407.80,
      "marketValue": 1400.00,
      "profitLoss": -7.80,
      "profitRate": -0.55
    }
  ]
}
```

### 2. 创建投资记录
```
POST /api/Investment
Content-Type: application/json

{
  "code": "005918",
  "name": "天弘沪深300ETF联接C",
  "type": "fund",
  "quantity": 1000,
  "costPrice": 1.4078,
  "notes": "支付宝购买"
}
```

### 3. 更新投资记录
```
PUT /api/Investment/{id}
Content-Type: application/json

{
  "code": "005918",
  "name": "天弘沪深300ETF联接C",
  "type": "fund",
  "quantity": 1000,
  "costPrice": 1.4078,
  "notes": "更新备注"
}
```

### 4. 删除投资记录
```
DELETE /api/Investment/{id}
```

### 5. 自动获取名称和价格
```
GET /api/Investment/auto-fill?code=005918&type=fund
```
**响应示例**：


## 🔄 工作流程

### 添加投资记录流程
1. 用户点击"新增投资"按钮
2. 填写代码（如：005918）
3. 系统自动识别类型（基金/股票）
4. 用户点击"自动获取"按钮（可选）
   - 系统调用 `/api/Investment/auto-fill` 接口
   - 后端调用东方财富 API 获取名称和价格
   - 如果获取成功，自动填充名称和当前价格
   - 如果获取失败，提示用户手动填写
5. 用户填写持仓数量和成本价
6. 用户点击"保存"按钮
7. 系统调用 `/api/Investment` POST 接口
8. 后端自动计算总成本、市值、盈亏、收益率
9. 保存到数据库
10. 刷新列表和统计数据

### 价格刷新流程
1. 用户点击"刷新价格"按钮
2. 系统调用 `/api/Investment/refresh-prices` 接口
3. 后端遍历所有投资记录
4. 对每条记录：
   - 根据代码和类型构建 secid 参数
   - 调用东方财富 API 获取最新价格
   - 更新当前价格、市值、盈亏、收益率
5. 保存到数据库
6. 刷新前端列表和统计数据

## 🔍 智能功能

### 1. 自动类型识别
根据代码自动识别是股票还是基金：
- **基金代码**：00、01、05、15、16、51-59 开头
- **股票代码**：0、3、6 开头（深市、创业板、沪市）

### 2. 自动价格获取
- 支持从东方财富 API 自动获取实时价格
- 如果 API 不支持（如场外基金），提示用户手动填写
- 自动将价格单位从"分"转换为"元"

### 3. 自动计算
- 保存时自动计算总成本、市值、盈亏、收益率
- 价格刷新时自动更新所有计算字段

## ⚠️ 注意事项

### 1. API 限制
- 东方财富 API 可能不支持所有基金代码（特别是场外基金）
- 如果自动获取失败，需要手动填写名称和价格
- API 调用有频率限制，建议不要过于频繁刷新

### 2. 数据准确性
- 当前价格来自第三方 API，可能存在延迟
- 建议在交易时间刷新价格
- 成本价需要用户手动输入，确保准确性

### 3. 错误处理
- API 调用失败时，系统会返回空数据，不会报错
- 用户可以手动填写所有信息，不依赖自动获取
- 所有错误都有友好的提示信息
