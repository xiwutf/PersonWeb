# 项目文档目录

本文档目录用于分类整理项目中的所有 Markdown 文档。

## ⭐ 必读文档

**在开始开发前，请务必阅读：**

- **[开发规范文档](./DEVELOPMENT_GUIDELINES.md)** ⭐ **必读** - 项目开发规范和要求
  - 样式管理规范（禁止内联样式）
  - 代码组织规范
  - 命名规范
  - 组件开发规范
  - API 开发规范
  - 数据库规范
  - Git 提交规范
  - 最佳实践和禁止事项

## 📁 文档分类

### 📋 功能开发文档 (`features/`)
功能开发相关的文档，包括功能状态、开发日志、优化计划等。

- `FEATURE_STATUS.md` - 功能开发状态报告
- `FEATURE_SUMMARY.md` - 功能开发状态总结
- `FEATURE_DEVELOPMENT_LOG.md` - 功能开发日志
- `OPTIMIZATION_PLAN.md` - 功能优化计划
- `CREATIVE_FEATURES.md` - 创意功能建议

### 🔧 改进与优化文档 (`improvements/`)
项目改进、优化相关的文档。

- `IMPROVEMENT_PLAN.md` - 改进计划
- `IMPROVEMENT_SUMMARY.md` - 改进总结
- `IMPROVEMENT_PROGRESS.md` - 改进进度
- `IMPROVEMENT_FINAL.md` - 最终改进报告
- `MARKDOWN_EDITOR_UPGRADE.md` - Markdown 编辑器升级文档

### ⚙️ 配置文档 (`config/`)
系统配置、API 配置、环境配置等相关文档。

- `CONFIG_SUMMARY.md` - 生产环境 API 配置总结
- `API_CONFIG.md` - API 配置说明
- `ENV_COMPATIBILITY.md` - 环境兼容性说明

### 🏗️ 架构文档 (`architecture/`)
系统架构、技术架构相关文档。

- `ARCHITECTURE.md` - 系统架构说明

### 🚀 部署文档 (`deployment/`)
部署、运维相关的文档。

- `DEPLOY.md` - 部署指南
- `START_BACKEND.md` - 后端启动指南

### 🐛 故障排除文档 (`troubleshooting/`)
问题排查、Bug 修复相关的文档。

- `TROUBLESHOOTING.md` - 通用故障排除指南
- `ARTICLES_TROUBLESHOOTING.md` - 文章系统故障排除
- `TOOLS_API_FIX.md` - 工具 API 修复记录
- `GIT_FIX.md` - Git 问题修复记录

### 📊 代码质量文档 (`quality/`)
代码质量、代码审查相关文档。

- `CODE_QUALITY_REPORT.md` - 代码质量报告

### 📝 迁移文档 (`migration/`)
数据迁移、系统迁移相关文档。

- `MIGRATION_TO_DATABASE.md` - 数据库迁移指南

### 📖 其他文档 (`misc/`)
其他杂项文档。

- `SLUG_GENERATION_EXPLAINED.md` - Slug 生成说明

## 📌 使用说明

1. **查找文档**：根据类别在对应目录下查找
2. **更新文档**：请在对应分类目录下更新
3. **新增文档**：根据内容选择合适的分类目录

## 🔄 文档维护

- 定期检查文档的时效性
- 删除过时或重复的文档
- 保持文档结构清晰
