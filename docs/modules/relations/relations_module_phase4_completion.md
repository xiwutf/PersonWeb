## 一、完成的功能

### ✅ 1.1 视图切换

**实现**：
- ✅ 在列表页顶部添加视图切换器（`n-segmented`）
- ✅ 四个视图选项：
  - **全部**：显示所有对象
  - **今天需要处理**：显示今天有提醒或需要关注的对象
  - **本周需要跟进**：显示本周需要跟进的对象（7天内未联系）
  - **看板视图**：按阶段分列的看板视图

**筛选逻辑**：

#### 今天需要处理
- 今天有提醒的（`remindAt` 是今天）
- 从未联系过的对象
- 7天以上未联系的对象

#### 本周需要跟进
- 本周有提醒的（7天内）
- 从未联系过的对象
- 7天以上未联系的对象

**显示**：
- 视图切换器下方显示当前视图的对象数量
- 根据视图类型显示相应的空状态提示

---

### ✅ 1.2 看板视图

**实现**：
- ✅ 按阶段（7个阶段）分列显示
- ✅ 每列显示：
  - 列标题（阶段名称 + 对象数量）
  - 该阶段的所有对象卡片
  - 空状态提示（如果该阶段没有对象）

**阶段列**：
1. 新认识 (0)
2. 熟悉中 (1)
3. 准备约见 (2)
4. 已见面 (3)
5. 升温中 (4)
6. 观察期 (5)
7. 已结束 (6)

**样式**：
- 横向滚动（响应式，小屏幕时）
- 每列固定最小宽度 320px
- 列内容可垂直滚动（最大高度限制）
- 阶段标签使用不同颜色区分

---

### ✅ 1.3 批量操作

**实现**：
- ✅ 看板视图时显示批量操作栏
- ✅ 对象卡片可选择（通过复选框）
- ✅ 批量操作按钮：
  - **批量标记观察期**：将选中的对象标记为"观察期"（阶段 5）
  - **批量标记已结束**：将选中的对象标记为"已结束"（阶段 6）
  - **清除选择**：清空所有选择

**选择功能**：
- 在看板视图的每个卡片右上角显示复选框
- 选中后卡片高亮显示（蓝色边框 + 阴影）
- 批量操作栏显示已选择的数量
- 批量操作前需要确认对话框

**操作流程**：
1. 切换到看板视图
2. 点击卡片上的复选框选择对象
3. 在批量操作栏点击操作按钮
4. 确认对话框后执行批量操作
5. 刷新列表，清除选择

### 2.2 看板视图布局

**实现**：
- 使用 `flex` 布局实现横向排列
- 每列使用 `flex: 1` 平均分配空间
- 响应式：小屏幕时横向滚动

**样式**：
```css

  flex: 1;
  min-width: 320px;
  display: flex;

  max-height: calc(100vh - 300px);

```

### 2.3 选择状态管理

**实现**：
- 使用 `Set<string>` 存储选中对象的 ID
- PersonCard 组件接收 `selected` 和 `selectable` props
- 通过 `@select` 事件更新选择状态

**代码**：
```typescript
const selectedPersons = ref<Set<string>>(new Set())

const handleSelectPerson = (id: string, selected: boolean) => {
  if (selected) {
    selectedPersons.value.add(id)
  } else {
    selectedPersons.value.delete(id)
  }
}
```
