# 关系跟进模块 - 功能描述文档

## 一、模块概述

这是一个**关系跟进（Dating CRM / Relationship Tracker）**模块，用于管理正在接触的对象与互动进展。目标用户是希望通过系统化方式管理社交关系的个人用户。

## 二、核心数据模型

### 1. 对象（RelationPerson）
- **基本信息**：昵称、来源（如：朋友介绍/社交软件/活动）、城市
- **分类标签**：标签（JSON 数组，如：["程序员","喜欢读书"]）
- **关系状态**：阶段（0=新认识, 1=熟悉中, 2=准备约见, 3=已见面, 4=升温中, 5=观察期, 6=已结束）
- **热度评分**：热度分数（0-100）
- **时间记录**：最后联系时间、最后见面时间、提醒时间
- **行动指导**：下一步行动、喜好/雷点、备注

### 2. 互动记录（RelationInteraction）
- **互动类型**：0=文字, 1=语音, 2=通话, 3=见面, 4=其他
- **发生时间**：occurredAt
- **内容摘要**：summary（必填）
- **关键信息**：keyPoints（JSON，存储要点/承诺/情绪等）
- **用户感受**：myFeeling（0=正面, 1=中性, 2=负面）
- **AI 建议**：aiSuggestion（AI 生成的分析建议）

### 3. 任务（RelationTask）
- **任务信息**：标题、截止时间、优先级（0=低, 1=中, 2=高, 3=紧急）
- **状态**：0=todo, 1=done, 2=hold
- **关联对象**：personId

## 三、当前功能实现

### 页面结构

#### 1. **列表页** (`/admin/relations`)
- **功能**：展示所有关系对象的卡片列表
- **筛选功能**：
  - 搜索：按昵称、来源搜索
  - 阶段筛选：按关系阶段筛选
  - 标签筛选：按标签筛选
  - 排序：按不同方式排序（最久未联系等）
- **操作**：
  - 新增对象按钮（顶部右侧）
  - 每个卡片提供快捷操作

#### 2. **对象卡片**（PersonCard 组件）
- **显示信息**：
  - 昵称、阶段标签
  - 来源、城市
  - 标签列表
  - 最后联系时间（相对时间：今天/昨天/X天前）
  - 热度分数
  - 下一步行动（高亮显示）
- **操作按钮**：
  - **查看详情**：跳转到详情页
  - **记录互动**：跳转到详情页的互动记录弹窗
  - **更多**：下拉菜单
    - 设提醒
    - 生成建议

#### 3. **详情页** (`/admin/relations/[id]`)
- **Tab 结构**：
  - **概览 Tab**：
    - 完整信息展示（所有字段的描述列表）
    - 编辑按钮（打开编辑弹窗）
  - **时间线 Tab**：
    - 展示所有互动记录（时间线形式）
    - 每条记录显示：时间、类型、摘要、关键点、感受、AI 建议
    - 操作：新增互动、编辑、删除
  - **任务 Tab**：
    - 任务列表（排序：未完成在前，优先级高的在前）
    - 操作：新增任务、编辑、删除、切换完成状态

#### 4. **弹窗组件**
- **AddPersonModal**：新增/编辑对象
- **AddInteractionModal**：新增/编辑互动记录
  - 包含 AI 生成建议功能（调用 AI 服务生成摘要、关键点、下一步行动、消息草案等）
- **AddTaskModal**：新增/编辑任务

### 核心功能

#### 1. **对象管理**
- 创建、编辑、删除对象
- 字段：昵称、来源、城市、标签、阶段、喜好/雷点、提醒时间、备注

#### 2. **互动记录管理**
- 创建、编辑、删除互动记录
- 记录互动类型、时间、摘要、关键点、感受
- **AI 辅助功能**：
  - 基于互动摘要和对象信息，AI 生成：
    - 一句话总结
    - 关键事实列表
    - 下一步行动建议（3条）
    - 消息草案（3条，不同场景）
    - 补充问题（引导用户补充信息）

#### 3. **任务管理**
- 创建、编辑、删除任务
- 设置优先级、截止时间
- 切换完成状态

#### 4. **提醒功能**
- 设置提醒时间（在对象编辑弹窗中）

#### 5. **筛选与搜索**
- 搜索昵称、来源
- 按阶段筛选
- 按标签筛选
- 排序（最久未联系等）

## 四、当前交互流程问题

### 问题描述

用户反馈：**"每个卡片的功能感觉都是跳转到了一个页面里"**

### 具体问题

1. **查看详情**：点击后跳转到详情页（新页面加载，打断当前浏览流程）
2. **记录互动**：点击后也跳转到详情页，需要在详情页再打开互动记录弹窗（操作路径长）
3. **快捷操作不够**：大部分操作都需要进入详情页，卡片上的快捷操作有限

### 用户期望

希望在**列表页的卡片上就能完成更多操作**，减少页面跳转，提升操作效率。

## 五、技术栈

- **前端**：Nuxt 3 + Vue 3 + TypeScript + Naive UI
- **后端**：.NET 8 WebAPI + EF Core + MySQL
- **AI 服务**：Python FastAPI（可选）
