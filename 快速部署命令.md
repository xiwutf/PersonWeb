# 阿里云ECS快速部署命令

## 📦 准备文件
✅ `website.zip` - 已生成（1.3MB）
✅ `阿里云ECS部署指南.md` - 详细文档

## 🚀 三步快速部署

### 第一步：确认ECS环境
```bash
# SSH连接到您的ECS服务器
ssh root@您的服务器IP

# 检查是否已安装Nginx
nginx -v

# 如果没有安装，运行：
sudo apt update && sudo apt install nginx -y  # Ubuntu
# 或者
sudo yum install epel-release -y && sudo yum install nginx -y  # CentOS
```

### 第二步：上传网站文件

#### 方案A：使用SCP上传压缩包（推荐）
```bash
# 在本地PowerShell中运行
scp website.zip root@您的服务器IP:/tmp/

# 然后在服务器上解压
ssh root@您的服务器IP
cd /tmp
unzip website.zip -d /var/www/html/
chown -R www-data:www-data /var/www/html/
chmod -R 755 /var/www/html/
```

#### 方案B：使用FTP工具
- 推荐：FileZilla、WinSCP
- 将 `website.zip` 上传到服务器的 `/tmp/` 目录
- 在服务器上解压到 `/var/www/html/`

### 第三步：配置Nginx

```bash
# 创建Nginx配置文件
cat > /etc/nginx/sites-available/personal-website << 'EOF'
server {
    listen 80;
    listen [::]:80;
    
    # 使用您的域名或者服务器IP
    server_name _;
    
    root /var/www/html;
    index index.html;
    
    location / {
        try_files $uri $uri/ /index.html;
    }
    
    location ~* \.(js|css|png|jpg|jpeg|gif|ico|svg|woff|woff2|ttf|eot)$ {
        expires 1y;
        add_header Cache-Control "public, immutable";
    }
    
    gzip on;
    gzip_types text/plain text/css application/json application/javascript text/xml application/xml;
}
EOF

# 启用配置
ln -sf /etc/nginx/sites-available/personal-website /etc/nginx/sites-enabled/
rm -f /etc/nginx/sites-enabled/default

# 测试并重启Nginx
nginx -t && systemctl reload nginx
```

## 🎯 一键部署脚本

将以下脚本保存为 `deploy.sh` 并在ECS服务器上运行：

```bash
#!/bin/bash
echo "🚀 开始部署个人网站..."

# 安装Nginx（如果未安装）
if ! command -v nginx &> /dev/null; then
    echo "📦 安装Nginx..."
    if command -v apt &> /dev/null; then
        apt update && apt install nginx -y
    elif command -v yum &> /dev/null; then
        yum install epel-release -y && yum install nginx -y
    fi
fi

# 创建网站目录
mkdir -p /var/www/html
cd /tmp

# 解压网站文件（假设website.zip已上传）
if [ -f "website.zip" ]; then
    echo "📂 解压网站文件..."
    unzip -o website.zip -d /var/www/html/
    chown -R www-data:www-data /var/www/html/ 2>/dev/null || chown -R nginx:nginx /var/www/html/
    chmod -R 755 /var/www/html/
fi

# 配置Nginx
echo "⚙️ 配置Nginx..."
cat > /etc/nginx/sites-available/personal-website << 'EOF'
server {
    listen 80;
    listen [::]:80;
    server_name _;
    root /var/www/html;
    index index.html;
    
    location / {
        try_files $uri $uri/ /index.html;
    }
    
    location ~* \.(js|css|png|jpg|jpeg|gif|ico|svg|woff|woff2|ttf|eot)$ {
        expires 1y;
        add_header Cache-Control "public, immutable";
    }
    
    gzip on;
    gzip_types text/plain text/css application/json application/javascript text/xml application/xml;
}
EOF

# 启用配置
mkdir -p /etc/nginx/sites-enabled
ln -sf /etc/nginx/sites-available/personal-website /etc/nginx/sites-enabled/
rm -f /etc/nginx/sites-enabled/default

# 启动Nginx
systemctl start nginx
systemctl enable nginx

# 配置防火墙
if command -v ufw &> /dev/null; then
    ufw allow 'Nginx Full' 2>/dev/null
elif command -v firewall-cmd &> /dev/null; then
    firewall-cmd --permanent --add-service=http 2>/dev/null
    firewall-cmd --permanent --add-service=https 2>/dev/null
    firewall-cmd --reload 2>/dev/null
fi

# 测试配置
if nginx -t; then
    systemctl reload nginx
    echo "✅ 部署完成！"
    echo "🌐 访问 http://$(curl -s ip.sb) 查看您的网站"
else
    echo "❌ Nginx配置有误，请检查"
fi
```

## 📱 阿里云安全组设置

确保在阿里云控制台开放以下端口：
- 入方向：80端口（HTTP）
- 入方向：443端口（HTTPS）
- 入方向：22端口（SSH）

路径：ECS控制台 → 实例 → 安全组 → 配置规则

## 🆘 常见问题

### 1. 无法访问网站
```bash
# 检查Nginx状态
systemctl status nginx

# 检查防火墙
iptables -L  # 或者 ufw status

# 查看错误日志
tail -f /var/log/nginx/error.log
```

### 2. 文件权限问题
```bash
chown -R www-data:www-data /var/www/html/  # Ubuntu
chown -R nginx:nginx /var/www/html/        # CentOS
chmod -R 755 /var/www/html/
```

### 3. 端口未开放
- 检查阿里云控制台安全组设置
- 确保80和443端口已开放
