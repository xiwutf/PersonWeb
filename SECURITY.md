## 报告安全漏洞

**请不要通过公开的 GitHub issue 报告安全漏洞。**

如果你发现了安全相关的问题，请发送电子邮件至 `your-email@example.com`。

请包括以下信息：

- 漏洞的描述
- 受影响的版本
- 复现步骤
- 潜在的影响
- 建议的修复（如果有）

我们会：

1. 在 24 小时内确认收到你的报告
2. 在 48 小时内评估漏洞
3. 在适当的情况下发布修复程序
4. 在修复发布后，在致谢中提及你（如果你希望）

## 安全最佳实践

### 对于使用者

1. **保持更新**: 定期更新依赖项
   ```bash
   npm update
   npm audit
   ```

2. **环境变量**: 不要在代码中提交敏感信息
   ```bash
   # 使用 .env 文件（不要提交到 git）
   cp .env.example .env
   ```

3. **依赖检查**: 定期检查已知漏洞
   ```bash
   npm audit
   npm audit fix
   ```

### 对于贡献者

1. **代码审查**: 新代码必须经过审查
2. **依赖**: 避免添加不必要的依赖
3. **版本**: 产品依赖使用固定版本
4. **验证**: 运行所有测试和安全检查

## 安全相关的依赖

本项目依赖于以下安全关键的库：

- Node.js 的加密模块（如适用）
- 任何用于身份验证的库

我们定期检查这些依赖的安全更新。
