# 🔒 网站安全说明

## 当前安全机制

### ✅ 网站是静态站点，内容只读

**重要说明**：您的网站是一个**静态网站**，这意味着：

1. **网站本身无法修改内容**
   - 网站只是展示静态文件（HTML、CSS、JS）
   - 没有任何后端API可以修改文件
   - 访问者只能查看，无法编辑

2. **内容存储在服务器文件系统中**
   - 所有内容文件（Markdown）存储在服务器的文件系统
   - 只有拥有服务器访问权限的人才能修改

3. **当前管理后台只是展示**
   - `/admin` 页面只是前端展示
   - 没有实际的文件写入功能
   - 任何人都可以访问，但无法修改内容

## 🔐 如何更新内容？

### 方式一：通过服务器文件系统（需要SSH权限）

```bash
# 1. SSH连接到服务器
ssh root@您的服务器IP

# 2. 编辑Markdown文件
nano /var/www/html/content/blog/my-article.md
# 或使用其他编辑器

# 3. 重新构建网站（如果使用动态生成）
cd /path/to/project
npm run build

# 4. 部署新文件
cp -r .output/public/* /var/www/html/
```

**安全性**：
- ✅ 需要SSH登录凭证
- ✅ 需要服务器文件系统访问权限
- ✅ 普通访问者无法执行

### 方式二：通过Git仓库（推荐）

```bash
# 1. 本地编辑文件
# 2. 提交到Git
git add content/blog/new-article.md
git commit -m "添加新文章"
git push origin master

# 3. 服务器上拉取更新
ssh root@服务器IP
cd /path/to/project
git pull origin master
npm run build
cp -r .output/public/* /var/www/html/
```

**安全性**：
- ✅ 需要Git仓库访问权限
- ✅ 需要SSH访问服务器
- ✅ 可以设置Git仓库为私有

### 方式三：构建后上传（最安全）

```bash
# 1. 本地构建
npm run build

# 2. 通过FTP/SCP上传
scp -r .output/public/* root@服务器IP:/var/www/html/
```

**安全性**：
- ✅ 完全在本地操作
- ✅ 需要FTP/SCP凭证
- ✅ 不暴露源代码

## 🛡️ 安全建议

### 1. 保护服务器访问权限

#### SSH安全配置

```bash
# 修改SSH默认端口
sudo nano /etc/ssh/sshd_config
# 修改 Port 22 为其他端口，如 Port 2222

# 禁用root登录（可选）
PermitRootLogin no

# 使用密钥认证
PasswordAuthentication no
PubkeyAuthentication yes

# 重启SSH服务
sudo systemctl restart sshd
```

#### 防火墙配置

```bash
# 只开放必要端口
sudo ufw allow 80/tcp   # HTTP
sudo ufw allow 443/tcp  # HTTPS
sudo ufw allow 22/tcp   # SSH（或您自定义的端口）
sudo ufw enable
```

### 2. 保护Git仓库

#### 设置私有仓库

- 在GitHub/GitLab等平台设置为私有仓库
- 只授权信任的人员访问
- 使用SSH密钥而非密码

#### 使用Git Hooks进行自动化部署

```bash
# 在服务器上创建post-receive hook
cd /path/to/git/repo.git/hooks
nano post-receive

# 添加部署脚本
#!/bin/bash
cd /path/to/project
git pull origin master
npm run build
cp -r .output/public/* /var/www/html/
```

### 3. 文件权限设置

```bash
# 设置正确的文件权限
# 网站文件：只读
chmod 644 /var/www/html/*.html
chmod 644 /var/www/html/**/*.html

# 目录：可执行
chmod 755 /var/www/html/

# 源代码目录：只有所有者可写
chmod 700 /path/to/project/
chown -R your-user:your-user /path/to/project/
```

### 4. 定期备份

```bash
# 创建备份脚本
#!/bin/bash
BACKUP_DIR="/backup/website"
DATE=$(date +%Y%m%d_%H%M%S)

# 备份网站文件
tar -czf $BACKUP_DIR/website_$DATE.tar.gz /var/www/html/

# 备份源代码
tar -czf $BACKUP_DIR/source_$DATE.tar.gz /path/to/project/

# 删除30天前的备份
find $BACKUP_DIR -name "*.tar.gz" -mtime +30 -delete
```

### 5. 监控和日志

```bash
# 监控文件变化
# 安装inotify-tools
sudo apt install inotify-tools

# 监控content目录
inotifywait -m -r /path/to/project/content/ -e modify,create,delete
```

## ⚠️ 当前存在的安全风险

### 1. 管理后台可公开访问

**问题**：`/admin` 页面任何人都可以访问

**解决方案**：

#### 方案A：添加IP白名单（Nginx配置）

```nginx
location /admin {
    allow 您的IP地址;
    deny all;
    
    # 其他配置...
}
```

#### 方案B：添加基础认证

```nginx
location /admin {
    auth_basic "Admin Area";
    auth_basic_user_file /etc/nginx/.htpasswd;
}
```

创建密码文件：
```bash
sudo apt install apache2-utils
sudo htpasswd -c /etc/nginx/.htpasswd admin
```

#### 方案C：完全隐藏管理后台

```nginx
# 重定向到404或首页
location /admin {
    return 404;
    # 或 return 301 /;
}
```

### 2. 源代码可能暴露

**问题**：如果部署时包含源代码

**解决方案**：
- ✅ 只部署 `.output/public` 目录
- ✅ 不要上传 `content/`、`pages/` 等源代码
- ✅ 使用 `.gitignore` 排除敏感文件

### 3. 服务器漏洞

**问题**：服务器系统可能存在安全漏洞

**解决方案**：
- 定期更新系统补丁
- 使用强密码
- 启用防火墙
- 安装安全监控工具

## 🔧 增强安全性的方案

### 方案一：添加真正的管理后台（需要后端）

如果需要在线编辑功能，需要：

1. **添加后端API**
   - 使用Node.js + Express
   - 或使用Nuxt Server API

2. **身份认证**
   - JWT Token认证
   - Session管理
   - 密码加密存储

3. **文件写入权限**
   - 限制文件写入路径
   - 文件类型验证
   - 内容安全检查

### 方案二：使用Headless CMS

- **Strapi**：开源Headless CMS
- **Contentful**：云端内容管理
- **Sanity**：实时协作CMS

这些方案提供：
- ✅ 安全的用户认证
- ✅ 权限管理
- ✅ API访问控制
- ✅ 内容审核

### 方案三：GitHub Actions自动部署

```yaml
# .github/workflows/deploy.yml
name: Deploy Website

on:
  push:
    branches: [ master ]

jobs:
  deploy:
    runs-on: ubuntu-latest
    steps:
      - uses: actions/checkout@v3
      - name: Setup Node.js
        uses: actions/setup-node@v3
      - name: Build
        run: npm install && npm run build
      - name: Deploy
        uses: easingthemes/ssh-deploy@main
        env:
          SSH_PRIVATE_KEY: ${{ secrets.SSH_PRIVATE_KEY }}
          REMOTE_HOST: ${{ secrets.REMOTE_HOST }}
          REMOTE_USER: ${{ secrets.REMOTE_USER }}
          SOURCE: ".output/public/"
          TARGET: "/var/www/html/"
```

**安全性**：
- ✅ 只有GitHub仓库有写权限的人才能触发
- ✅ 使用SSH密钥认证
- ✅ 自动构建和部署

## 🆘 如果发现安全问题

1. **立即更改所有密码**
2. **检查服务器日志**：`/var/log/auth.log`、`/var/log/nginx/access.log`
3. **检查文件完整性**：对比备份文件
4. **更新所有软件**：`sudo apt update && sudo apt upgrade`
5. **联系服务器提供商**：报告安全事件
