# 📝 内容更新指南

本指南介绍如何更新网站内容（博客文章、项目、工具插件）。

---

## 🎯 更新方式概览

项目支持以下几种内容更新方式：

1. **本地编辑 + Git 推送**（推荐，适合开发者）
2. **直接编辑 Markdown 文件**（简单直接）
3. **使用在线编辑器**（需要额外配置）
4. **自动化部署流程**（CI/CD）

---

## 📋 方式一：本地编辑 + Git 推送（推荐）

### 适用场景
- 熟悉 Git 和代码编辑器
- 需要版本控制
- 团队协作

### 操作步骤

#### 1. 更新博客文章

```bash
# 1. 克隆或拉取最新代码
git pull origin master

# 2. 编辑或创建新文章
# 在 content/blog/ 目录下创建或编辑 .md 文件
# 例如：content/blog/my-new-post.md

# 3. 提交更改
git add content/blog/my-new-post.md
git commit -m "添加新文章：我的新文章"
git push origin master
```

#### 2. 更新工具插件

```bash
# 编辑 content/tools/ 目录下的文件
# 例如：content/tools/revit-plugin-1.md

git add content/tools/revit-plugin-1.md
git commit -m "更新工具：批量族加载器"
git push origin master
```

#### 3. 更新项目信息

```bash
# 编辑 content/projects/ 目录下的文件
git add content/projects/finance-helper.md
git commit -m "更新项目状态：智能理财助手"
git push origin master
```

## 📋 方式二：直接编辑 Markdown 文件（最简单）

### 适用场景
- 快速更新内容
- 不熟悉 Git


- 页面会自动热更新
- 无需重启服务器
- 立即看到效果

#### 4. 构建并部署

```bash

# 例如：上传 .output/public 目录到服务器
```

## 📋 方式四：自动化部署（CI/CD）

### GitHub Actions 自动部署

创建 `.github/workflows/deploy.yml`：

```yaml
name: Deploy Website

on:
  push:
    branches: [ master ]
  pull_request:
    branches: [ master ]

jobs:
  build-and-deploy:
    runs-on: ubuntu-latest
    
    steps:
    - uses: actions/checkout@v3
    
    - name: Setup Node.js
      uses: actions/setup-node@v3
      with:
        node-version: '18'
    
    - name: Install dependencies
      run: npm install
    
    - name: Build
      run: npm run build
    
    - name: Deploy to server
      uses: easingthemes/ssh-deploy@main
      env:
        SSH_PRIVATE_KEY: ${{ secrets.SSH_PRIVATE_KEY }}
        REMOTE_HOST: ${{ secrets.REMOTE_HOST }}
        REMOTE_USER: ${{ secrets.REMOTE_USER }}
        SOURCE: ".output/public/"
        TARGET: "/var/www/html/"
```

### 配置说明

1. 在 GitHub 仓库设置中添加 Secrets：
   - `SSH_PRIVATE_KEY`：服务器 SSH 私钥
   - `REMOTE_HOST`：服务器 IP 或域名
   - `REMOTE_USER`：SSH 用户名

2. 每次推送代码到 master 分支，自动：
   - 安装依赖
   - 构建项目
   - 部署到服务器

## 🎨 内容管理最佳实践

### 1. 文件命名规范

- ✅ 使用小写字母和连字符：`my-article.md`
- ❌ 避免空格和特殊字符：`我的文章.md`（不推荐）

### 2. 日期格式

- 使用 ISO 格式：`2024-01-20`
- 保持一致性

### 3. 图片管理

- 将图片放在 `public/images/` 目录
- 使用相对路径：`/images/blog/cover.jpg`
- 优化图片大小（建议 < 500KB）

### 4. 标签和分类

- 标签：使用数组格式 `["Vue.js", "教程"]`
- 分类：保持一致，建议使用固定分类列表

### 5. 内容组织

```
content/
├── blog/
│   ├── 2024-01-20-my-article.md
│   └── 2024-01-15-another-article.md
├── tools/
│   └── revit-plugin-1.md
└── projects/
    └── my-project.md
```

## 🚨 常见问题

### Q1: 更新后网站没有变化？

**A:** 检查以下几点：
1. 文件是否保存在正确目录
2. Front Matter 格式是否正确
3. 是否重新构建了项目
4. 浏览器缓存是否清除

### Q2: 图片显示不出来？

**A:** 
- 确认图片路径使用 `/images/` 前缀
- 检查图片文件是否存在于 `public/images/` 目录
- 确认文件扩展名正确

### Q3: 如何删除内容？

**A:** 
- 直接删除对应的 `.md` 文件
- 重新构建项目
- 或者将文件移动到其他目录（保留备份）

### Q4: 如何批量更新？

**A:** 
- 使用代码编辑器的批量查找替换功能
- 编写脚本批量处理（Node.js/Python）
- 使用 Git 进行版本管理
